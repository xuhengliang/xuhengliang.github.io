Redis入门

概念

一个基于内存的NoSQL数据库
NoSQL(Not Only SQL):
不仅仅是SQL,代表非关系型数据

分类:

四大类:键值型,列簇型,文档型,图文型

特点:

1.因为数据是存放在内存中,因此redis读写能力是很快的
2.redis又有别于其他的键值型数据库,redis提供了磁盘的持久化方案(RDB和AOF),对数据的安全提供了一定的保证
3.对比传统的关系型数据库来说,还是关系型数据库安全性会更强一些,所以通常reddis和普通的关系型数据(mysql,oracle…)协同合作

运用场景:

数据缓存服务器-最常见的应用场景

基于redis读写快速的特点,通常在项目结构中,充当缓存服务器的角色,用来缓存项目中的热点数据,当用户访问热点数据的时候,不用再去访问数据库了,直接从redis中就可以获得数据,极大的提高了请求的响应速度

应对高速读写的场合

对于某些并发量很大的场景,并且业务逻辑也很复杂的场景,这个时候将数据库临时更新到redis中
比如:秒杀,先把数据存放到redis中,然后对redis进行读写操作,在高速读写的时间段过去后,再利用定时器将redis中的数据写回到MySQL中

数据共享

在分布式集群环境下,有些数据如果放在session中,那么会导致其他的tomcat服务器无法访问这个数据,这样用户就会发现数据的展示时有时无;通常来说,这些数据就需要进行共享处理,redis就可以充当数据共享的服务器
比如:用户信息

分布式锁

1.基于redis底层是单线程
2.实现:setNX与Lua脚本保证原子性操作实现
3.集群环境下,某些同步业务方法,无法再有效的保证业务安全,因为当前工程的同步关键字不能锁住其他工程的线程,因此完全有可能有几个线程进行相关的业务操作;这时候,他们可以通过redis分布式锁让不同的工程区同一个地方加锁,从而保证只有一个线程在进行业务操作,保证线程安全

Redis基本命令

数据结构分类:

String(字符串)
常用命令:

1.set key value : 设值
2.mset key1 value1 key2 value2… : 设置多个值
3.get key : 取值
4.mget key1 key2… : 取多个值
5.del key1 key2… : 删除键值对
6.strlen key : 长度
7.getset key value : 先获得再设置值(修改值),会返回显示原有值
8.getrange key start end (含头含尾) : 截取从start到end的部分值
9.append key value : 追加

运算操作:

1.incr key : 自增
2.incrby key n : 按n自增
3.decr key : 自减
4.decrby key n : 按n自减

代码操作
Hash(哈希)-更符合Java对象的存储

1.hset key field value : 设值
2.hget key field : 取值
3.hmset key field1 value1 field2 value2… : 设置多个字段的值
4.hmget key field1 field2… : 获取多个字段的值
5.hdel key field field2… : (del是直接删除key) 删除字段
6.hexists key field : 判断字段是否存在
7.hlen key : 返回key中field的个数
8.hkeys key : 返回所有field
9.hvals key : 返回所有的value
10.hgetall : 返回所有field和value

代码:
List(链表)

1.lpush key node1 node2… : 向左压入node
2.rpush key node1 node2… : 向右压入node
3.lindex key index : 获得左边index的node的值
4.llen key : node个数
5.lpop key : 弹出最左边的node
6.rpop key : 弹出最右边的node
7.linsert key before|after oldNode newNode : 选择插入到node的前后
8.lrange key start end : 取指定start到end的node
9.lset key index node : 替换指定index下标的node
10.ltrim key start end : 去除除了start到end的其他node

######链表的阻塞命令:
1.blpop key timeout
2.brpop key timeout
3.rpoplpush key src : 将右边的node弹出然后压到右边
4.brpoplpush src timeout

Set(集合)
特点

1.集合采用的是哈希表结构
2.集合的数据是不能重复的
3.集合是无序的
4.集合的每个元素都是String数据结构类型

基本命令

1.sadd key value1 value2… : 添加
2.scard key : 获得集合元素的数量
3.sdiff key1 key2 [store dest] : 取差集(存到第三个集合)
4.sinner key1 key2 [store dest] : 取交集
5.sunion key1 key2 [store dest] : 取并集
6.smembers key : 查看所有元素
7.srem key value1 value2… : 删除

代码
ZSet(有序集合)

1.zadd key score1 value1 score2 values2… : 添加元素+评分
2.zcard key : 返回集合中元素的个数
3.zrange key start stop : 按照分数的大小返回成员，start和stop指定范围
4.zcount key min max : 返回指定范围内分数的成员数量
5.zincrby key n value : 为集合成员value增加分数n
